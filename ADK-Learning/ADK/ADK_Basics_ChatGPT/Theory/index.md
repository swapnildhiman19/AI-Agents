I think this will be much stronger

At the end of this mini-course, I don't just want you to know how to write:

instruction="Hello {name}"

I want you to be able to answer questions like:

Why does ADK have a Session at all?
Why isn't state just a global Python dictionary?
Why are artifacts separate from state?
Why is there a Runner instead of calling the agent directly?
What problem does an Event solve?
At what point is {name} replaced with its value?
What exactly is sent to Gemini?
If something goes wrong, which component is responsible?

When you can answer those, you'll understand ADK as a system rather than as a collection of classes. That's the level of understanding that makes learning the rest of the framework much easier. I think this foundation is well worth the extra chapter before we write more code.
